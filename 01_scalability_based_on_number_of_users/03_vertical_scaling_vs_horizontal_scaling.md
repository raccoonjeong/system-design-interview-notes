## 수직적 규모 확장 vs 수평적 규모 확장

- 스케일 업(scale up)
  - 수직적 규모 확장(vertical scailing)
  - 서버에 고사양 자원(더 좋은 CPU, 더 많은 RAM 등)을 추가
  - 장점
    - 트래픽의 양이 적을 때 적합
    - 단순함
  - 단점
    - 한계가 있음. 한 대의 서버에 자원을 무한대로 증설할 수 없음
    - 장애에 대한 자동복구(failover) 방안이나 다중화(redundancy) 방안을 제시하지 않음

- 스케일 아웃(scale out)
  - 수평적 규모 확장
  - 더 많은 서버를 추가
  - 대규모 애플리케이션에 보다 적절

### 로드밸런서

부하 분산 집합(load balancing set)에 속한 웹 서버들에게 트래픽 부하를 고르게 분산하는 역할

- 보안을 위해 사설 주소 이용

- 부하 분산 집합에 웹 서버 추가시 장점
  - 장애 자동복구(failover) 가능
  - 웹 계층의 가용성 향상
  - 서버 1이 다운되면 모든 트래픽은 2로 전송됨, 따라서 웹 사이트 전체 다운되는 일 방지
  - 두 대로 트래픽이 감당안될 때, 서버를 추가하기만 됨. 로드밸런서가 자동으로 트래픽을 분산함

### 데이터베이스 다중화

- 주(master) 데이터베이스: 쓰기 연산(insert, delete, update) 지원
- 부(slave) 데이터베이스: 주에서부터 사본 전달받아 읽기 연산 지원

대부분의 애플리케이션은 읽기 연산의 비중이 쓰기 연산보다 훨씬 높으므로, 주 보다 부의 수가 많다.

- 다중화의 장점
  - 더 나은 성능
    - 병렬로 처리되는 질의(query)의 수가 늘어나므로, 성능이 좋아짐
  - 안정성
    - 자연 재해에도 데이터 보존
  - 가용성
    - 하나의 DB서버에 장애 발생해도 다른 서버에 있는 데이터를 가져와 계속 서비스할 수 있음

* 가용성(Availability)은 시스템, 서비스, 정보 등이 사용자가 필요로 하는 시점에 얼마나 안정적이고 원활하게 사용 가능한 상태를 유지하는가를 뜻하며, 정보 보안의 3대 요소(기밀성, 무결성, 가용성) 중 하나로, 시스템의 가동률이나 접근 가능성을 나타내는 척도입니다. 이는 '필요할 때 언제든지 사용할 수 있는 능력'으로, 높을수록 시스템이 안정적임
