## 로그, 메트릭 그리고 자동화

사업 규모가 커지면 로그, 메트릭, 자동화에 필수적으로 투자해야 한다.

- 로그
  - 에러 로그 모니터링은 중요함. 시스템의 오류와 문제들을 쉽게 찾아낼 수 있도록 하기 때문에
  - 에러 로그는 서버 단위로 모니터링 할 수도 있지만, 단일 서비스로 모아주는 도구를 활용하면 더 편리하게 검색, 조회 가능

- 메트릭
  - 메트릭을 잘 수집하면 사업 현황에 관한 유용한 정보를 얻을 수 있고, 시스템의 현재 상태 쉽게 파악 가능
  - 특히 유용한 것
    - 호스트 단위 메트릭
      - CPU, 메모리, 디스크I/O에 관한 메트릭
    - 종합(aggregated) 메트릭
      - 데이터베이스 계층의 성능, 캐시 계층의 성능
    - 핵심 비즈니스 메트릭
      - 일별 능동 사용자(daily active user), 수익(revenue), 재방문(retention)

- 자동화
  - 시스템이 커지면 생산성을 높이기 위해 도구를 활용해야 함
  - ex) 지속적 통합(continuous integration)을 도와주는 도구
    - 검증, 빌드, 테스트, 배포 등의 절차를 자동화해서 개발 생산성을 크게 향상시킬 수 있음
