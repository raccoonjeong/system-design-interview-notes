## 데이터 센터

- 지리적 라우팅(geoDNS-routing / geo-routing)
  - 장애가 없는 상황에서 사용자가 가장 가까운 데이터 센터로 안내되는 것
  - geoDNS는 사용자의 위치에 따라 도메인 이름을 어떤 IP 주소로 변환할지 결정해주는 서비스

- 다중 데이터센터 아키텍처
  - 장애 발생시 장애가 없는 데이터 센터로 전송됨

- 다중 데이터센터 아키텍처를 만들 때 생기는 기술적 난제
  - 트래픽 우회
    - 올바른 데이터 센터로 트래픽을 보내는 효과적인 방법을 찾아야 함. ex) GeoDNS - 가장 가까운 데이터센터로 보냄
  - 데이터 동기화(synchroniztion)
    - 보편적 전략은 여러 데이터 센터에 걸쳐 데이터를 다중화함
 - 테스트와 배포
   - 애플리케이션을 여러 위치에서 테스트해야 한다.
   - 자동화된 배포 도구는 모든 데이터 센터에 동일하게 서비스를 설치하게 한다.
